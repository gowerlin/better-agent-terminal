## A. 語音錄音與 RecordingService（8 項）

> 來源：T0005（PromptBox 語音 UI）+ T0003（IPC 基礎設施）

- [ ] **測試 A.1**：正常錄音 3~5 秒 → 辨識 → 文字填入 textarea
  - **步驟**：1. 確認已下載 small 模型 2. 點麥克風按鈕 3. 說話 3~5 秒 4. 再點按鈕停止 5. 預覽框出現 → 按確認
  - **預期**：辨識結果（繁中）正確填入 PromptBox textarea
  - **失敗條件**：textarea 無文字、文字為空白、文字為簡體
  - **相關工單**：T0005, T0004

- [ ] **測試 A.2**：連續兩次錄音（state reset）
  - **步驟**：1. 完成一次錄音+確認 2. 立即再次點麥克風按鈕 3. 說話並停止 4. 確認第二次結果
  - **預期**：第二次辨識結果正確 append 到 textarea，hook state 完全 reset
  - **失敗條件**：第二次錄音無法開始、結果覆蓋第一次文字、state 殘留
  - **相關工單**：T0005

- [ ] **測試 A.3**：錄音中 Alt+M 停止
  - **步驟**：1. 按 Alt+M 開始錄音 2. 說話 2 秒 3. 按 Alt+M 停止
  - **預期**：與點按鈕停止行為一致，進入辨識流程
  - **失敗條件**：Alt+M 無反應、錄音未停止、狀態卡住
  - **相關工單**：T0005

- [ ] **測試 A.4**：模型未下載時按麥克風按鈕
  - **步驟**：1. 確認無已下載模型（刪除全部）2. 點麥克風按鈕
  - **預期**：按鈕為 disabled 狀態 + tooltip 提示需先下載模型
  - **失敗條件**：按鈕可點擊、無提示訊息、開始錄音
  - **相關工單**：T0005

- [ ] **測試 A.5**：拒絕麥克風權限
  - **步驟**：1. 點麥克風按鈕 2. 在 Chromium 權限對話框點「封鎖」
  - **預期**：顯示明確錯誤訊息（RecordingError code='permission-denied'），不 crash
  - **失敗條件**：無錯誤訊息、app crash、靜默失敗
  - **相關工單**：T0005, T0003

- [ ] **測試 A.6**：極短錄音 <0.5 秒
  - **步驟**：1. 點麥克風按鈕 2. 立即再點停止（<0.5 秒）
  - **預期**：graceful 處理（可能回傳空結果或短音訊提示），不 crash
  - **失敗條件**：app crash、whisper 錯誤未被捕捉、state 卡住
  - **相關工單**：T0005

- [ ] **測試 A.7**：切換 terminal tab 後回來（isActive prop）
  - **步驟**：1. 開始錄音 2. 切換到另一個 terminal tab 3. 切回原 tab
  - **預期**：isActive=false 時 Alt+M 不觸發，切回後狀態正確
  - **失敗條件**：非 active tab 觸發 Alt+M、切回後 state 不一致
  - **相關工單**：T0005

- [ ] **測試 A.8**：PromptBox 既有功能 runtime 回歸
  - **步驟**：1. 測試 slash command autocomplete（輸入 `/`）2. 測試圖片貼上 3. 測試 command history（上下鍵）4. 測試送出按鈕
  - **預期**：所有既有功能正常運作，不受語音 UI 影響
  - **失敗條件**：任一既有功能失效或行為改變
  - **相關工單**：T0005

## C. 預覽框 Popover 流程（10 項）

> 來源：T0007（語音預覽框 popover）

- [ ] **測試 C.1**：錄音 → 停止 → popover 自動顯示 loading
  - **步驟**：1. 開始錄音 2. 說話 3 秒 3. 停止
  - **預期**：popover 從 MicButton 上方出現，顯示「辨識中...」loading 動畫
  - **失敗條件**：popover 未出現、無 loading 狀態
  - **相關工單**：T0007

- [ ] **測試 C.2**：辨識完成 → popover 顯示結果文字
  - **步驟**：接續 C.1，等待辨識完成
  - **預期**：loading 消失，顯示繁中辨識結果文字，可在 textarea 中編輯
  - **失敗條件**：仍顯示 loading、無結果文字、textarea 不可編輯
  - **相關工單**：T0007

- [ ] **測試 C.3**：編輯辨識結果後確認
  - **步驟**：1. 辨識完成後，在 popover textarea 修改文字 2. 按 ✓ 確認
  - **預期**：**修改後**的文字填入 PromptBox textarea，popover 消失
  - **失敗條件**：填入的是修改前的原文、popover 未消失
  - **相關工單**：T0007

- [ ] **測試 C.4**：取消辨識結果
  - **步驟**：1. 辨識完成後，按 ✗ 取消
  - **預期**：文字丟棄，popover 消失，PromptBox textarea 不變
  - **失敗條件**：文字被填入 textarea、popover 未消失
  - **相關工單**：T0007

- [ ] **測試 C.5**：Enter 確認快捷鍵
  - **步驟**：1. 辨識結果顯示在 popover 中 2. 按 Enter
  - **預期**：等同點 ✓ 確認（capture phase 優先於 PromptBox 送出）
  - **失敗條件**：Enter 觸發 PromptBox 送出而非 popover 確認
  - **相關工單**：T0007

- [ ] **測試 C.6**：Esc 取消快捷鍵
  - **步驟**：1. 辨識結果顯示在 popover 中 2. 按 Esc
  - **預期**：等同點 ✗ 取消（capture phase 優先於 PromptBox 清空）
  - **失敗條件**：Esc 清空 PromptBox textarea 而非只取消 popover
  - **相關工單**：T0007

- [ ] **測試 C.7**：Shift+Enter 在預覽框 textarea 換行
  - **步驟**：1. 辨識結果顯示在 popover 中 2. 在 textarea 按 Shift+Enter
  - **預期**：textarea 換行（不觸發確認）
  - **失敗條件**：觸發確認、無法換行
  - **相關工單**：T0007

- [ ] **測試 C.8**：錯誤狀態 popover 顯示
  - **步驟**：1. 觸發辨識錯誤（例：模型檔案損毀或被刪除）
  - **預期**：popover 顯示錯誤訊息，同時 hint 區域也顯示（雙重回饋）
  - **失敗條件**：無錯誤顯示、只有單一處顯示
  - **相關工單**：T0007

- [ ] **測試 C.9**：popover 位置穩定性
  - **步驟**：1. 在不同視窗大小下觸發 popover 2. 觀察位置
  - **預期**：popover 始終出現在 MicButton 上方（CSS 相對定位）
  - **失敗條件**：popover 位置偏移、被裁切
  - **相關工單**：T0007

- [ ] **測試 C.10**：連續兩次錄音 → 預覽框狀態正確 reset
  - **步驟**：1. 完成第一次錄音+確認 2. 立即開始第二次錄音+停止
  - **預期**：第二次 popover 正確顯示 loading → 結果，無殘留狀態
  - **失敗條件**：popover 顯示上次結果、狀態混亂
  - **相關工單**：T0007
