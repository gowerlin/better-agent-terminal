# 語音輸入功能 — 使用者測試指引

> **版本**：Phase 1 Final  
> **產出日期**：2026-04-11  
> **測試項目總數**：32  
> **分 Part 數**：7  
> **適用對象**：不需看程式碼即可依步驟完成所有測試

---

## 開始前準備

### 環境需求

- **Windows 10/11**（主要測試環境）
- macOS 10.14+（選擇性測試）
- 可用的麥克風（內建或外接）
- 網路連線（首次下載語音模型時需要）
- 建議至少 2 GB 可用磁碟空間（small 模型 465 MB）

### 啟動 App

1. 開啟終端，cd 到專案目錄
2. 執行 `npm run dev`
3. Electron 視窗啟動後即可開始測試

### 若失敗如何回報

每個測試項目如果失敗，請記錄以下資訊：
1. **測試編號**（例如 1.1）
2. **失敗現象**（看到什麼，與預期有何不同）
3. **螢幕截圖**（如果方便的話）
4. **作業系統和 DPI 設定**

回報到專案 Issue 或直接告知開發者。

## Part 1：初次設定（3 個測試）

### 測試 1.1：下載語音模型

1. 點右上角齒輪圖示開啟 **Settings** 面板
2. 向下滾動找到「**🎤 語音輸入**」區塊
3. 看到 4 個語音模型列表：tiny / base / small / medium
4. 點「**Small（推薦）**」旁的「下載」按鈕
5. 觀察進度條，等待下載完成（約 465 MB，視網速 1~5 分鐘）

**預期**：下載完成後「Small」狀態變為「已下載」，radio 可選

**若失敗**：檢查網路連線。若進度條卡住不動，嘗試點「取消」後重新下載。回報「測試 1.1 失敗：進度條卡在 XX%」。

### 測試 1.2：確認預設設定

1. 下載完成後觀察 Settings 語音區塊
2. 檢查以下預設值：
   - 預設模型：Small
   - 預設語言：繁中（zh）
   - 「自動轉繁體」開關：ON

**預期**：三項預設值正確

**若失敗**：回報「測試 1.2 失敗：XXX 設定為 YYY，預期 ZZZ」。

### 測試 1.3：首次麥克風權限

1. 關閉 Settings 面板
2. 在 PromptBox（底部輸入框）找到**麥克風按鈕** 🎤
3. 點按鈕
4. 系統會彈出 Chromium 層的麥克風權限對話框
5. 點「**允許**」

**預期**：
- **Windows**：Chromium 權限對話框出現，允許後按鈕變紅色 + pulse 動畫（表示正在錄音）
- **macOS**：系統權限對話框出現 → 允許後開始錄音

**若失敗**：若無對話框出現，檢查瀏覽器/系統是否已經封鎖過麥克風。回報「測試 1.3 失敗：未出現權限對話框」。

## Part 2：核心語音流程（6 個測試）

### 測試 2.1：基本錄音與辨識

1. 點麥克風按鈕開始錄音（按鈕變紅 + pulse 動畫）
2. 對麥克風說 3~5 秒中文，例如：「你好，這是一個測試」
3. 再點一次麥克風按鈕停止錄音
4. 觀察 MicButton 上方出現**預覽框 popover**
5. 預覽框先顯示「辨識中...」loading
6. 等待 1~5 秒（視模型和電腦速度）
7. 辨識完成後，預覽框顯示繁體中文結果

**預期**：預覽框顯示正確的繁體中文辨識結果

**若失敗**：回報辨識結果（是空白？亂碼？簡體？英文？）以及等待多久。

### 測試 2.2：Alt+M 快捷鍵

1. 確認 PromptBox 所在的 terminal tab 是 active
2. 按 **Alt+M** 開始錄音
3. 說話 2~3 秒
4. 再按 **Alt+M** 停止

**預期**：與點按鈕行為完全一致

**若失敗**：回報「Alt+M 無反應」或「Alt+M 觸發了其他功能」。

### 測試 2.3：連續錄音

1. 完成測試 2.1 並確認結果（文字填入 textarea）
2. 不清空 textarea，**立即**再次點麥克風按鈕
3. 說另一段話，例如：「這是第二次測試」
4. 停止 → 等辨識 → 確認

**預期**：第二段文字 **append** 到第一段後面，不覆蓋

**若失敗**：回報「第二次無法錄音」或「覆蓋了第一次結果」。

### 測試 2.4：取消辨識結果

1. 錄音 + 停止 → 等辨識結果出現在預覽框
2. 按預覽框的 **✗ 取消按鈕**（或按 **Esc** 鍵）

**預期**：預覽框消失，PromptBox textarea 不變（辨識文字被丟棄）

**若失敗**：回報「文字仍被填入」或「popover 未消失」。

### 測試 2.5：編輯辨識結果

1. 錄音 + 停止 → 等辨識結果出現在預覽框
2. 在預覽框的 textarea 中**手動修改**文字（例如加上「修改過的：」前綴）
3. 按 **✓ 確認**（或按 **Enter** 鍵）

**預期**：填入 PromptBox 的是**修改後**的文字，不是原始辨識結果

**若失敗**：回報「填入的是修改前的原文」。

### 測試 2.6：辨識錯誤處理

1. 點麥克風按鈕
2. **不說話**，在完全安靜的環境下等 1 秒
3. 停止錄音

**預期**：graceful 處理 — 可能回傳空結果或提示「無法辨識」，不 crash

**若失敗**：回報「app crash」或「永遠顯示 loading」。

## Part 3：Settings 頁面（5 個測試）

### 測試 3.1：模型切換

1. 下載 tiny 模型（最小，77.7 MB）
2. 在 Settings 把預設模型切換為 tiny
3. 關閉 Settings → 錄音 → 觀察辨識速度

**預期**：辨識速度比 small 快（tiny 較小但品質較低），偏好已儲存

**若失敗**：回報「無法切換模型」或「切換後辨識仍使用 small」。

### 測試 3.2：取消下載

1. 開始下載 base 模型（141 MB）
2. 在進度約 30% 時點「取消」

**預期**：下載中斷，進度條消失，模型狀態回到「未下載」

**若失敗**：回報「取消按鈕無反應」或「暫存檔仍在磁碟上」。

### 測試 3.3：刪除已下載模型

1. 找到已下載的 tiny 模型
2. 點「刪除」按鈕
3. 確認刪除對話框

**預期**：模型狀態變「未下載」，磁碟空間釋放

**若失敗**：回報「刪除後狀態未更新」或「檔案仍存在」。

### 測試 3.4：語言切換

1. 在 Settings 語音區塊找到語言選擇
2. 切換為 **English**
3. 錄音說一段英文
4. 觀察辨識結果

**預期**：辨識結果為英文

**若失敗**：回報「仍辨識為中文」。

### 測試 3.5：開啟模型資料夾

1. 點「開啟資料夾」按鈕

**預期**：系統檔案總管開啟 whisper-models 目錄，可看到已下載的 `.bin` 模型檔案

**若失敗**：回報「按鈕無反應」或「開啟錯誤目錄」。

## Part 4：BUG-001 驗證 — 貼上修復（3 個測試）

> BUG-001：Claude OAuth 登入時 paste 被截斷的問題  
> 修復方式：改用 xterm.js 的 `terminal.paste()` 方法支援 bracketed paste

### 測試 4.1：Shell 指令貼上

1. 開一個 terminal tab
2. 複製以下指令：
   ```
   echo "Hello World" && echo "Test paste" && echo "Line 3"
   ```
3. 在 terminal 按 **Ctrl+V** 貼上

**預期**：指令完整貼入，按 Enter 後三行 echo 都正確輸出

**若失敗**：回報「指令被截斷到第 X 個字元」。

### 測試 4.2：TUI 貼上（vim/nano）

1. 在 terminal 執行 `nano test.txt`（或 `vim test.txt` 按 `i` 進入插入模式）
2. 複製一段 5 行以上的多行文字
3. 按 Ctrl+V 貼上

**預期**：文字完整貼入，無自動縮排混亂（bracketed paste markers 正確送出）

**若失敗**：回報「每行都被自動縮排」或「只貼了部分文字」。

### 測試 4.3：Claude /login 完整授權流程（**最重要**）

1. 在 terminal 執行 `claude`
2. 輸入 `/login`
3. 複製畫面上的 OAuth URL 到系統瀏覽器
4. 在瀏覽器完成登入，取得授權碼
5. 回到 terminal，在 `Paste code here if prompted >` 欄位按 **Ctrl+V** 貼授權碼

**預期**：授權碼**完整**傳入（不只是尾端幾個字元），登入成功

**若失敗**：回報「只貼了 X 個字元」或「登入失敗 - invalid code」。這是 BUG-001 的原始症狀。

**預期**：首次使用時 macOS 系統權限對話框正確顯示，授權後語音流程正常

**若失敗**：回報 macOS 版本和具體異常。

## Part 7：邊界情境（8 個測試）

### 測試 7.1：極短錄音

1. 點麥克風按鈕
2. **立即**（<0.5 秒）再點停止

**預期**：graceful 處理 — 回傳空結果或提示太短，不 crash

**若失敗**：回報「app crash」或「state 卡住」。

### 測試 7.2：較長錄音（30 秒）

1. 點麥克風按鈕
2. 持續說話 30 秒
3. 停止

**預期**：辨識時間較長但最終正確回傳結果

**若失敗**：回報「超時」或「記憶體錯誤」。

### 測試 7.3：離線辨識（斷網）

1. 確認模型已下載
2. **斷開網路**
3. 錄音 + 辨識

**預期**：辨識正常（whisper 是完全離線引擎）

**若失敗**：回報「辨識失敗」及錯誤訊息。

### 測試 7.4：模型切換後立即錄音

1. 在 Settings 切換預設模型（例如 small → tiny）
2. 關閉 Settings
3. 立即開始錄音

**預期**：使用新選的模型辨識

**若失敗**：回報「仍使用舊模型」。

### 測試 7.5：拒絕麥克風權限後的行為

1. 若可以重置 Chromium 麥克風權限（或使用 incognito-like 環境）
2. 點麥克風按鈕
3. 在權限對話框點「封鎖」

**預期**：顯示明確錯誤訊息，提示需要允許麥克風權限

**若失敗**：回報「無錯誤提示」或「app crash」。

### 測試 7.6：popover 開啟時切換 tab

1. 錄音 + 停止 → popover 出現
2. 在 popover 開啟狀態下**切換到另一個 terminal tab**

**預期**：popover 正確處理（消失或保持，不 crash）

**若失敗**：回報「popover 殘留在錯誤位置」或「crash」。

### 測試 7.7：PromptBox 既有功能回歸

1. 在 textarea 輸入 `/`，觀察 slash command autocomplete
2. 複製一張圖片，Ctrl+V 貼到 textarea
3. 按上下鍵測試 command history
4. 點送出按鈕

**預期**：所有既有功能正常，不受語音 UI 影響

**若失敗**：回報「XXX 功能失效」。

### 測試 7.8：繁簡轉換開關效果

1. Settings 中「自動轉繁體」設為 ON
2. 錄音中文 → 確認結果為繁體
3. Settings 中「自動轉繁體」設為 OFF
4. 錄音中文 → 確認結果為簡體

**預期**：開關正確控制繁簡轉換

**若失敗**：回報「開關無效」或「兩次結果相同」。
