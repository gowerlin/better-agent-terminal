# .private/ 目錄說明

此目錄用於存放**不應進入 git repo 的塔台敏感內容**。

## 使用時機
- 存放使用者個人思考草稿
- 敏感資料（客戶資訊、內部基礎設施等）
- 未經整理的討論記錄

## Git 狀態
`.private/` 整個目錄被 `.gitignore` 排除（見 `.gitignore` 塔台目錄規則）。

## 如何使用
1. 建立 `_ct-workorders/.private/` 目錄
2. 將任何敏感內容放入此目錄
3. Git 不會追蹤此目錄內任何檔案

## 跨機器同步
由於 `.private/` 不進 repo，跨機器時需另行同步（例如手動 copy 或使用 Syncthing）。
