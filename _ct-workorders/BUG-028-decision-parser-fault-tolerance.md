# BUG-028 — 決策日誌 Parser 容錯不足，跨專案顯示不完整

## 狀態
**🚫 CLOSED**（T0119 修復，人工驗收通過 2026-04-16）

## 發現時間
2026-04-15 21:20 UTC+8

## 嚴重度
**🟡 Medium**（顯示不完整，不影響核心功能）

## 現象描述

CT 面板「決策」頁籤在部分專案只顯示決策 ID（D001、D002…），沒有標題和摘要。

**已確認受影響專案**：
- `D:\ForgejoGit\2025_Gogoro`（塔台 v4 產生的決策日誌格式）

**截圖**：決策卡片只有 D001~D005 ID，無標題、無日期、無摘要內容。統計列顯示「📌 5 / 5 條」正確，但 card body 空白。

## 預期行為

每張決策卡片應顯示：
- D### ID
- 日期
- 標題摘要
- 展開後完整內容

## 根因推測

決策日誌 Parser（`_decision-log.md` 解析器）對格式的預設假設過嚴格。塔台 v4 升級流程產生的 `_decision-log.md` 使用了與原始手動格式不同的結構（如標題行格式、索引表結構），Parser 無法正確擷取 title / date / content。

## 修復方向

1. **調查**：比對 `better-agent-terminal` 和 `2025_Gogoro` 兩個專案的 `_decision-log.md` 格式差異
2. **容錯**：Parser 應支援多種決策日誌格式（v3 手動版 + v4 自動產生版）
3. **Fallback**：解析失敗時至少顯示原始文字內容，不應靜默吞掉

## 影響範圍

- CT 面板 → 決策頁籤（DecisionLogView 或同等元件）
- 所有使用 Control Tower v4 的專案都可能受影響
